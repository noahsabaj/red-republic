# Red Republic — Planned Economy Builder

Isometric city-builder (React 19 + Vite 7 + TypeScript, canvas renderer). Package manager is **bun** (`bun.lock` is the lockfile — never commit a `package-lock.json`).

## Commands

- `bun run dev` — dev server on port 3000 (`?demo` seeds a developed town, `?seed=N` reproduces a specific map). The port is strict — kill stray servers first.
- `bun run check` — the full gate: `tsc -b && eslint . && vitest run`. Run before committing.
- `bun run test` — Vitest only. **Never `bun test`** — Bun's native runner grabs `*.test.ts` with incompatible semantics.
- `bun run build` — production build (tsc + vite)
- `bun run desktop:dev` — native Tauri window with HMR (starts its own vite on :3000; a browser tab can attach to the same server). `bun run desktop:build` — local installers under `src-tauri/target/release/bundle/`.

## Architecture

- `src/game/engine.ts` — `GameEngine`: single mutable sim class, fixed-timestep day loop (`advance(dtMs)`, 500 ms = 1 day at 1× speed). All randomness flows through the seeded RNG created from `engine.seed` (deterministic per seed — keep it that way; no bare `Math.random()` in sim code; `mulberry32` exposes `getState()/setState()` so saves restore streams bit-exact). Constructed from `{seed, mapW, mapH, climate, difficulty, name}`.
- `src/game/config.ts` — all game data/balance (buildings, resources, prices, objectives, `CLIMATES`, `MAP_SIZES`, `DIFFICULTIES` — difficulty is start-conditions only; the sim is difficulty-blind). Data lives here, behavior in the engine.
- `src/game/mapgen.ts` — variable map sizes (32–128 per axis; presets in `MAP_SIZES`). Feature densities scale with area but **48×48 output must stay byte-identical per seed** (same rnd() call order — `mapgen-snapshot.test.ts` is the tripwire; never repin it to make a change pass). Engine/render/pathfind read dimensions from `engine.mapW/mapH`, never from constants.
- `src/game/weather.ts` — `WeatherTimeline(seed, climate)`: deterministic climate per seed (temperature + Markov conditions + snow depth + river freeze), generated sequentially and memoized so `engine.weather` and the exact `forecast()` share one source. Numeric climates live in config's `CLIMATES`; the default `plains` preset is pinned by `climate.test.ts` and must never drift.
- `src/game/save-format.ts` + `src/game/save-slots.ts` — versioned save blobs (`engine.serialize()` / `GameEngine.fromSave()`, packed tile codec, `SaveError`) and the localStorage slot store (manual slots, `quicksave`, rotating autosaves, export/import). The engine never imports the slot store. Entity payloads hydrate over a defaults spread — a new engine field with a sensible default needs no format bump; anything that can't be defaulted bumps `SAVE_FORMAT_VERSION` with a `MIGRATIONS` step. Loads always come back paused.
- `src/game/pathfind.ts` — multi-source BFS floods over the road network on reusable generation-stamped scratch buffers, lazily resized to the requested map (a resize bumps the generation — stale views must throw, not read zeroed stamps). A `FloodResult` is a view valid only until the next flood; `snapshot()` before caching.
- `src/game/input.ts` — DOM-free pointer/keyboard gesture state machine (pan/paint/pinch/select, WASD/edge panning, Escape = cancel tool, else open menu). `GameCanvas.tsx` is a thin Pointer-Events adapter around it; live preferences arrive via the injected `InputOptions` provider.
- `src/game/render.ts` — isometric canvas renderer. Draw order is the row scan: anything world-positioned (buildings, trucks, citizens, placement ghost) must draw at its row inside the scan, never as a post-scan overlay, or it will float over buildings that should occlude it. Status colors come from `STATUS_PALETTES` (default/colorblind); `UIState.reducedMotion` freezes decoration (particles, glints, bob) but never sim-driven movement.
- `src/app/` — the meta-game shell: `screens.ts` (pure reducer over menu/pause screens — exactly one dialog can exist at a time; Escape is a single `BACK` semantic), `session.ts` (`GameSession` = engine + founding config + monotonic id; the id keys `<GameCanvas>` so new-game/load remounts camera+input+debug hook — never swap engines in place), `settings.ts` (typed localStorage store with engine-style subscribe; React reads it via `use-settings.ts`, canvas/input/audio read `getSettings()` live). `?demo`/`?seed=N`/`?climate=` bypass the menu (dev workflow).
- `src/audio/` — procedural WebAudio (zero assets): synth SFX + generative minor-key score. **Audio listens, never participates**: it hooks UI callbacks and the single App event drain (`drainEvents()` is destructive — one consumer, fanned out), reads an engine probe, and must never import engine internals or touch sim RNG. Context creation is gesture-gated (`audio.unlock()`).
- React bridge: the engine `bump()`s a version counter; components subscribe via the hooks in `src/hooks/use-engine.ts` (signature selectors for cheap components, full-version for detail panels). `App.tsx` itself does not subscribe globally.
- `src/components/ui/` and `src/hooks/use-mobile.ts` are vendored shadcn/ui — excluded from linting, don't hand-edit. Menu screens live in `src/components/menu/` (hand-rolled Tailwind on the shared `MenuShell`; inline `TwoStepButton` confirms, no nested modals).

## Rules

- **UI never re-computes simulation math.** Panels display engine APIs — `productionRates()`, `importPriceOf()`, `sellableStock()` — and mutate via engine methods (`toggleStaffPriority()`), never by writing engine fields. This is what keeps display and simulation from diverging.
- **One owner for persistent UI surfaces.** Every right-side view is a `PanelMode` owned by `App` (exclusive by construction). Never give a component its own open/close state for a persistent surface — that's how the stockpiles popover ended up overlaying the Five-Year Plan.
- **No emoji; icons come from `src/ui/icons.ts`.** One Lucide-backed registry renders as SVG (`<GameIcon>`) and on canvas (`drawIcon`). config/engine reference icons by name string; `ui-guards.test.ts` fails the build on unknown names, emoji in first-party sources, or invalid Tailwind color-opacity steps (e.g. `bg-red-950/97` silently emits no CSS).
- **Weather is engine state.** `engine.weather` / `engine.forecast()` are the only sources of truth — never derive weather from `season()` or the calendar in UI or render code (heating is temperature-driven via `heatingRequired()`, river ice follows `weather.riverFrozen`). Gameplay multipliers live in config's `WEATHER` table; the timeline draws from its own decorrelated RNG stream so it never perturbs economy randomness.
- **Money is foreign currency only.** Nothing domestic costs rubles or dollars: buildings and roads cost materials + builder labor (roads/bridges are real per-tile construction sites — gravel or planks+steel trucked in, undrivable until complete, `becomesRoad` dissolves the site into a road tile), and there are no wages. Money enters at the border (exports, contracts, Moscow's objective rewards) and leaves at the border (imports, contract fines) — the sole domestic money sink is instant build, priced by `engine.instantCost()` as an imported Western prefab. Machinery is the physical face of that dependency: industry needs it to build AND to run (`wear` drains per-building spare bins daily; a dry bin means `buildingWorn()` and ×`wornEffMult` output, never a stall) — imported at first, produced domestically once a Machine Works closes the loop. `campaign-pacing.test.ts` is the balance tripwire (a pinned 3-year build order in `campaign.ts` asserting solvency + the machinery arc): if a deliberate rebalance moves the curve, re-derive the trajectory with a scratch runner and re-pin — never loosen thresholds blindly.
- **Trade crosses the border physically.** One map edge per seed is the national border (`tile.foreign`, `engine.borderEdge`); foreign soil is unbuildable and customs houses must stand on it, each clearing a staffed daily throughput (`BALANCE.customsThroughputPerDay × eff`). Auto-trade policy, the daily `tradeLedger` and `contracts` are engine state mutated only via engine methods (`setAutoTradeRule()`, `acceptContract()`, …); auto-exports are truck-staged to customs stock before sale, and both sale paths pay through `exportPayout()` so contract deliveries can't be missed. Contract offers draw from a stateless per-month rng stream — never from the economy stream.
- **Saves must round-trip bit-exact.** `save-roundtrip.test.ts` serializes, reloads, then advances both engines 90 days and compares everything — a new mutable engine field must be added to `serialize()`/`fromSave()` (and hydrates over a default) or that test is the tripwire that catches you.
- **The web build is primary** — the browser workflow must never break; desktop is a Tauri shell around the same dist. `src/platform/` is the ONLY layer that touches `@tauri-apps` APIs (always via `isTauri()`-gated dynamic imports — the web bundle stays Tauri-free) or raw `localStorage`: all persistence goes through `storage()` from `src/platform/storage.ts` (browser = localStorage, desktop = real files in `%APPDATA%\com.noahsabaj.redrepublic\` via an in-memory cache with async write-behind — `save-roundtrip`-style sync callers keep working, disk failures surface on the error bus). Any desktop quit path must call `quitApp()` (flushes pending writes, then destroys the window) — never `getCurrentWindow().close()`, which loops the close-confirm. `src-tauri/` is Rust, not covered by `bun run check`; capabilities follow least privilege. The app version lives in `package.json` only (`tauri.conf.json` points at it). Release = bump version → tag `vX.Y.Z` → push → CI drafts a GitHub Release with installers → wait for the run, verify the Windows assets, then **publish without pausing** (`gh release edit vX.Y.Z --draft=false --latest`; user directive: "always ship") — publishing arms the auto-updater for every installed copy.
- Every bug fix lands with a regression test in `src/game/__tests__/`. Engine tests build worlds with `helpers.ts` (`makeEngine()` = flat map, no starting base, **no border**, **calm weather** — pass `weather:` to script conditions; `flatBorderMap()` opts into border rules; real-map behavior needs `new GameEngine` directly; `flatMap(w, h)` takes dimensions). Remember `pop` clamps to housing capacity at day end — place beds before setting `pop`.
- Dev builds expose `window.__redRepublic = { engine, cam }` for console debugging and automated verification (bound per GameCanvas mount — absent while on the main menu).
