# Red Republic ‚Äî Planned Economy Builder

Isometric city-builder (React 19 + Vite 7 + TypeScript, canvas renderer). Package manager is **bun** (`bun.lock` is the lockfile ‚Äî never commit a `package-lock.json`).

## Commands

- `bun run dev` ‚Äî dev server on port 3000 (`?demo` seeds a developed town, `?seed=N` reproduces a specific map). The port is strict ‚Äî kill stray servers first.
- `bun run check` ‚Äî the full gate: `tsc -b && eslint . && vitest run`. Run before committing.
- `bun run test` ‚Äî Vitest only. **Never `bun test`** ‚Äî Bun's native runner grabs `*.test.ts` with incompatible semantics.
- `bun run build` ‚Äî production build (tsc + vite)
- `bun run desktop:dev` ‚Äî native Tauri window with HMR (starts its own vite on :3000; a browser tab can attach to the same server). `bun run desktop:build` ‚Äî local installers under `src-tauri/target/release/bundle/`.

## Architecture

- `src/game/engine.ts` ‚Äî `GameEngine`: single mutable sim class, fixed-timestep day loop (`advance(dtMs)`, 500 ms = 1 day at 1√ó speed). All randomness flows through the seeded RNG created from `engine.seed` (deterministic per seed ‚Äî keep it that way; no bare `Math.random()` in sim code; `mulberry32` exposes `getState()/setState()` so saves restore streams bit-exact). Constructed from `{seed, mapW, mapH, climate, difficulty, name}`.
- `src/game/config.ts` ‚Äî all game data/balance (buildings, resources, prices, objectives, `CLIMATES`, `MAP_SIZES`, `DIFFICULTIES` ‚Äî difficulty is start-conditions only; the sim is difficulty-blind). Data lives here, behavior in the engine.
- `src/game/mapgen.ts` ‚Äî variable map sizes (32‚Äì128 per axis; presets in `MAP_SIZES`). Feature densities scale with area but **48√ó48 output must stay byte-identical per seed** (same rnd() call order ‚Äî `mapgen-snapshot.test.ts` is the tripwire; never repin it to make a change pass). Engine/render/pathfind read dimensions from `engine.mapW/mapH`, never from constants.
- `src/game/weather.ts` ‚Äî `WeatherTimeline(seed, climate)`: deterministic climate per seed (temperature + Markov conditions + snow depth + river freeze), generated sequentially and memoized so `engine.weather` and the exact `forecast()` share one source. Numeric climates live in config's `CLIMATES`; the default `plains` preset is pinned by `climate.test.ts` and must never drift.
- `src/game/save-format.ts` + `src/game/save-slots.ts` ‚Äî versioned save blobs (`engine.serialize()` / `GameEngine.fromSave()`, packed tile codec, `SaveError`) and the localStorage slot store (manual slots, `quicksave`, rotating autosaves, export/import). The engine never imports the slot store. Entity payloads hydrate over a defaults spread ‚Äî a new engine field with a sensible default needs no format bump; anything that can't be defaulted bumps `SAVE_FORMAT_VERSION` with a `MIGRATIONS` step. Loads always come back paused.
- `src/game/pathfind.ts` ‚Äî multi-source weighted shortest-path floods on reusable generation-stamped scratch buffers, lazily resized to the requested map (a resize bumps the generation ‚Äî stale views must throw, not read zeroed stamps). `floodCost()` is a bucket-queue (Dial's) Dijkstra over small integer entry costs (maxStep 1 ‚áí plain FIFO BFS with hop-count distances); `shortestPathToAny()` is the bounded counterpart that stops at the nearest ranked goal. `maxStep` is validated (`Number.isInteger`, `‚â§ MAX_SUPPORTED_STEP`) *before* any buffer/gen side effect, and the topology owns it: `TopologyIndex.maxStep(domain)` is the max entry cost the domain's own mask produced, so the pathfinder bound can never disagree with the costs (road/water ‚Üí 1, land ‚Üí offRoadStepCost). `FloodResult.distanceAt` returns accumulated travel **cost**; a view is valid only until the next flood ‚Äî consume it before flooding again.
- `src/game/input.ts` ‚Äî DOM-free pointer/keyboard gesture state machine (pan/paint/pinch/select, WASD/edge panning; keys: Escape = cancel tool else open menu, **X = arm bulldoze**, Space = pause, 1/2/3 = speed). `GameCanvas.tsx` is a thin Pointer-Events adapter around it; live preferences arrive via the injected `InputOptions` provider.
- `src/game/render.ts` ‚Äî isometric canvas renderer. Draw order is the row scan: anything world-positioned (buildings, trucks, citizens, placement ghost) must draw at its row inside the scan, never as a post-scan overlay, or it will float over buildings that should occlude it. Status colors come from `STATUS_PALETTES` (default/colorblind); `UIState.reducedMotion` freezes decoration (particles, glints, bob) but never sim-driven movement.
- `src/app/` ‚Äî the meta-game shell: `screens.ts` (pure reducer over menu/pause screens ‚Äî exactly one dialog can exist at a time; Escape is a single `BACK` semantic), `session.ts` (`GameSession` = engine + founding config + monotonic id; the id keys `<GameCanvas>` so new-game/load remounts camera+input+debug hook ‚Äî never swap engines in place), `settings.ts` (typed localStorage store with engine-style subscribe; React reads it via `use-settings.ts`, canvas/input/audio read `getSettings()` live). `?demo`/`?seed=N`/`?climate=` bypass the menu (dev workflow).
- `src/audio/` ‚Äî procedural WebAudio (zero assets): synth SFX + generative minor-key score + unified interaction sound. **Audio listens, never participates**: it hooks UI callbacks and the single App event drain (`drainEvents()` is destructive ‚Äî one consumer, fanned out), reads an engine probe, and must never import engine internals or touch sim RNG. Context creation is gesture-gated (`audio.unlock()`). **One catalog, one dispatcher, one owner per moment** ‚Äî three channels feed the `audio` singleton and nothing double-fires: press (a `click`/keypress ‚Üí `audio.ui(family)`), outcome (a result ‚Üí `audio.sfx(name)`), engine event (`onGameEvent`). Press sound has a single owner: a capture-phase delegated listener (`installUiSounds`, mounted once in `App`) classifies any activated control via the pure, node-testable `classify()` in `ui-catalog.ts` (`ElementDesc` descriptor, never a live DOM node ‚Äî the test env is `node`, no jsdom) and plays a mechanical **in-key** voice from `UI_VOICES` (bodies pitched to the score's live chord via `clickHz`+`MusicEngine.currentChord()`; noise stays unpitched). Controls opt out with `data-sfx="none"` when their result/funnel already sounds (trade, build-tool arm); keyboard nav calls `uiSound(family)` ‚Äî same dispatcher. Three buses (music/effects/**interface**) with independent volumes; menu families ride interface, world cues (`select`/`toolArm`/`toolCancel`) ride effects (`FAMILY_BUS`/`SFX_BUS`). Adding a control needs no wiring ‚Äî untagged buttons infer `neutral`; `data-sfx` only upgrades semantics.
- React bridge: the engine `bump()`s a version counter; components subscribe via the hooks in `src/hooks/use-engine.ts` (signature selectors for cheap components, full-version for detail panels). `App.tsx` itself does not subscribe globally.
- `src/components/BottomBar.tsx` ‚Äî the gameplay build UI (a **bottom bar**, W&R-inspired, replaced the old left `BuildMenu`): a 3-tier drill-down (category ‚Üí sub-category ‚Üí building grid from config's `CATEGORIES`/`SUBCATEGORIES`), docked placement-mode toggles, and a right-end **demolish + hide/pin** cluster (pinned by default; retracts to a sliver you hover to peek ‚Äî the retract uses an inline `style` transform, not a Tailwind arbitrary `calc()` class, which silently emits nothing). The toggles set the default `BuildPolicy` (`{autoBuy, currency, instant, plan}`) stamped onto each new site; per-site editing happens on the `SidePanel` detail panel. Checkbox controls are gone ‚Äî the reusable pressed `ToggleButton` (`menu/controls.tsx`, `aria-pressed`+`data-sfx="toggle"`) is the on/off idiom everywhere.
- `src/components/ui/` and `src/hooks/use-mobile.ts` are vendored shadcn/ui ‚Äî excluded from linting, don't hand-edit. Menu screens live in `src/components/menu/` (hand-rolled Tailwind on the shared `MenuShell`; inline `TwoStepButton` confirms, no nested modals).

## Rules

- **UI never re-computes simulation math.** Panels display engine APIs ‚Äî `productionRates()`, `importPriceOf()`, `sellableStock()` ‚Äî and mutate via engine methods (`toggleStaffPriority()`), never by writing engine fields. This is what keeps display and simulation from diverging.
- **The economy is bulk/continuous; wholeness is an edge property, not a representation.** `b.stock[r]` is a float by design (production/wear/consumption are all fractional rates) and that is correct ‚Äî every resource is bulk (tonnage), none discrete. "Whole units" matter only at three edges: trade, contracts, and display. The border already trades whole units (`sell`/`buy`/auto-trade floor); the **display** edge is the `src/game/format.ts` vocabulary ‚Äî `fmtQty` (whole amount), `fmtLevel` (precise bin gauge ‚Äî the one place a fraction is shown, on purpose), `fmtRate` (per-day flow), `fmtOwed` (shortfall ‚Äî ceils, the mirror of `fmtQty`'s floor), `fmtMoney`, `fmtPct`. Never format a player-facing quantity with ad-hoc `.toFixed`/`Math.floor` (that scatter is what leaked the `42.14 coal` float tail); `ui-guards.test.ts` fails the build on a raw quantity beside a resource name in an engine event. A genuinely discrete good (whole-unit inventory, discrete failure) would be a localized `ResourceDef.kind: 'discrete'` + a banking accumulator for that one resource ‚Äî deliberately **not** built (no uniform flag while every good is bulk).
- **One owner for persistent UI surfaces.** Every right-side view is a `PanelMode` owned by `App` (exclusive by construction). Never give a component its own open/close state for a persistent surface ‚Äî that's how the stockpiles popover ended up overlaying the Five-Year Plan.
- **No emoji; icons come from `src/ui/icons.ts`.** One Lucide-backed registry renders as SVG (`<GameIcon>`) and on canvas (`drawIcon`). config/engine reference icons by name string; `ui-guards.test.ts` fails the build on unknown names, emoji in first-party sources, or invalid Tailwind color-opacity steps (e.g. `bg-red-950/97` silently emits no CSS).
- **Weather is engine state.** `engine.weather` / `engine.forecast()` are the only sources of truth ‚Äî never derive weather from `season()` or the calendar in UI or render code (heating is temperature-driven via `heatingRequired()`, river ice follows `weather.riverFrozen`). Gameplay multipliers live in config's `WEATHER` table; the timeline draws from its own decorrelated RNG stream so it never perturbs economy randomness.
- **Money is foreign currency only.** Nothing domestic costs rubles or dollars *for free labor*: buildings and roads cost materials + builder labor (roads/bridges are real per-tile construction sites ‚Äî gravel or planks+steel trucked in, undrivable until complete, `becomesRoad` dissolves the site into a road tile). Money enters at the border (exports, contracts, Moscow's objective rewards) and leaves at the border (imports, contract fines). Construction policy is **per-site** ‚Äî the bottom-bar toggles stamp a default `BuildPolicy` onto each new site, editable afterward on its `SidePanel` (`setSiteImport`/`setSiteForeignLabor`/`finishSiteInstant`/`commenceSite`). Three domestic ‚ÇΩ/$ sinks, all border-priced: **instant build/finish** ($, `instantCost`/`instantFinishCost` ‚Äî a Western prefab at placement, or completing an existing site's *remaining* work); **auto-buy** (‚ÇΩ East **or** $ West, `autoBuyImportCost(defId,x,y,currency)`/`autoBuyRemainingCost` ‚Äî pays a site's import bill, delivered as **bonded virtual imports** earmarked via a `{from: customsId, bonded: true}` demand that never touches customs stock ‚Äî cap-exempt, leak-proof); and **foreign construction labor** (‚ÇΩ/builder-day, `BALANCE.foreignLaborPerDay`) ‚Äî `construction()` is **two-phase**: citizens build every ready site free, then paid foreign builders top up only sites whose per-site `foreignLabor` permits it (default = the global `foreignLaborEnabled` menu toggle), affordability-capped so ‚ÇΩ never goes negative, surfaced on `tradeLedger.*.foreignLabor` (the single-pass progress step keeps it byte-identical to the old lump when every site allows foreign labor). **Planning mode**: a `plan`-placed site is `paused` (orders nothing, no builders) until `commenceSite()`/`commenceAllPlanned()`, which charges any deferred auto-buy at commence. Machinery is the physical face of that dependency: industry needs it to build AND to run (`wear` drains per-building spare bins daily; a dry bin means `buildingWorn()` and √ó`wornEffMult` output, never a stall) ‚Äî imported at first, produced domestically once a Machine Works closes the loop. `campaign-pacing.test.ts` is the balance tripwire (a pinned 3-year build order in `campaign.ts` asserting solvency + the machinery/foreign-labor arc): if a deliberate rebalance moves the curve, re-derive the trajectory with a scratch runner and re-pin ‚Äî never loosen thresholds blindly.
- **Roads are the fast freight network, not a hard gate.** Trucks prefer roads but may drive **off-road** across any non-water, non-foreign, unoccupied land tile at `BALANCE.offRoadStepCost`√ó the per-tile time, so a building with no road still receives slow deliveries (dispatch in `logistics()` tries a road-only bounded search first, a weighted-land one second via `routeToSupply` ‚Äî road-connected towns stay byte-identical; `floodTerrain` still backs the off-road refund). Water still needs bridges/ports/barges; off-road is land-only. `b.connected` = a finite-cost land-or-road path reaches the depot network (gates staffing/builders/trucks); `b.roadConnected` is the stricter road-only state driving the "off-road ‚Äî slow; lay a road" advisory. Bulldozing an unfinished site returns its **already-delivered** `stock` to storage by truck (`refundSiteStock`, off-road capable) ‚Äî never let it vanish; the in-flight turn-back conserves cargo still on the road, the two are disjoint.
- **Trade crosses the border physically.** One map edge per seed is the national border (`tile.foreign`, `engine.borderEdge`); foreign soil is unbuildable and customs houses must stand on it, each clearing a staffed daily throughput (`BALANCE.customsThroughputPerDay √ó eff`). Auto-trade policy, the daily `tradeLedger` and `contracts` are engine state mutated only via engine methods (`setAutoTradeRule()`, `acceptContract()`, ‚Ä¶); auto-exports are truck-staged to customs stock before sale, and both sale paths pay through `exportPayout()` so contract deliveries can't be missed. Contract offers draw from a stateless per-month rng stream ‚Äî never from the economy stream.
- **Saves must round-trip bit-exact.** `save-roundtrip.test.ts` serializes, reloads, then advances both engines 90 days and compares everything ‚Äî a new mutable engine field must be added to `serialize()`/`fromSave()` (and hydrates over a default) or that test is the tripwire that catches you.
- **The web build is primary** ‚Äî the browser workflow must never break; desktop is a Tauri shell around the same dist. `src/platform/` is the ONLY layer that touches `@tauri-apps` APIs (always via `isTauri()`-gated dynamic imports ‚Äî the web bundle stays Tauri-free) or raw `localStorage`: all persistence goes through `storage()` from `src/platform/storage.ts` (browser = localStorage, desktop = real files in `%APPDATA%\com.noahsabaj.redrepublic\` via an in-memory cache with async write-behind ‚Äî `save-roundtrip`-style sync callers keep working, disk failures surface on the error bus). Any desktop quit path must call `quitApp()` (flushes pending writes, then destroys the window) ‚Äî never `getCurrentWindow().close()`, which loops the close-confirm. The user-facing exits ‚Äî main-menu **Exit Game** and the pause **Exit ‚Üí (Main Menu / Desktop)** chooser ‚Äî are desktop-only (gated on `isTauri()` via an optional prop / `canQuit`, never an inline `isTauri()` in render) and route desktop-exit through `quitApp()`; web keeps a single Exit-to-Main-Menu. `src-tauri/` is Rust, not covered by `bun run check`; capabilities follow least privilege. The app version lives in `package.json` only (`tauri.conf.json` points at it). Release = bump version ‚Üí tag `vX.Y.Z` ‚Üí push ‚Üí CI drafts a GitHub Release with installers ‚Üí wait for the run, verify the Windows assets, then **publish without pausing** (`gh release edit vX.Y.Z --draft=false --latest`; user directive: "always ship") ‚Äî publishing arms the auto-updater for every installed copy.
- Every bug fix lands with a regression test in `src/game/__tests__/`. Engine tests build worlds with `helpers.ts` (`makeEngine()` = flat map, no starting base, **no border**, **calm weather** ‚Äî pass `weather:` to script conditions; `flatBorderMap()` opts into border rules; real-map behavior needs `new GameEngine` directly; `flatMap(w, h)` takes dimensions). Remember `pop` clamps to housing capacity at day end ‚Äî place beds before setting `pop`.
- **Toolbar Toggle Button Invariance:** Toggle buttons (`ToggleButton`) in toolbar rows must maintain a fixed label and fixed icon, expressing ON vs OFF strictly through active/inactive color states (`on` prop) to prevent layout shifts and visual jumping across state changes.
- **Master Switch Overrides:** Global master switches (`globalConstructionEnabled`, `foreignLaborEnabled`) must act as non-destructive simulation gates without mutating per-entity site flags (`b.paused`, `b.foreignLabor`), ensuring individual user preferences are preserved when resumed.
- Dev builds expose `window.__redRepublic = { engine, cam }` for console debugging and automated verification (bound per GameCanvas mount ‚Äî absent while on the main menu).
